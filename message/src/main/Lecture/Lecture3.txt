메시지, 국제화
    프로젝트 설정

    메시지, 국제화 소개
        다양한 메시지를 한곳에서 관리하도록 하는 기능을 메시지기능이라한다.
        메시지 관리용 파일을 만들고 html에서 해당데이터를 key값으로 호출하여 사용하는것

        국제화는 이 메시지 관리용 파일을 나라별로 별도로 관리하면 그것이 국제화이다.

        HTTP accept-language 해더값을 사용하거나 사용자가 직접언어선택 쿠키등을 사용해서 처리하면 된다.

        메시지, 국제화를 직접 구현할 수 있지만 스프링은 이를 지원하는 기능을 제공한다.
        타임리프 또한 이와 관련되서 편리한 기능 제공 어떤 기능???

    스프링 메시지 소스 설정
        메시지 소스 기본 값
        spring.messages.basename=messages
        messages라는 이름으로 기본 등록된다 따라서 우리는 messages.properties, messages_en.properties파일만 등록하면 자동으로 인식된다.

    스프링 메시지 소스 사용
        메시지를 못찾으면 디폴트메시지를 준다.

    웹 애플리케이션에 메시지 적용하기
        타임 리프의 메시지 표현식 #{ ... } 를 사용하면 스프링의 메시지를 편하게 조회가능!

    웹 애플리 케이션에 국제화 적용하기
        메시지 기능은 Locale 정보를 알아야 언어 선택 가능

        header 설정 뿐만 아니라 쿠키나 세션등을 통해서도 Locale을 변경 할 수 있도록 해줄 수 있다.
        LocaleResolver를 사용하면 된다.

    정리
        유연하게 설정해놓으면 유지/보수에 쉽다 메시지기능을 잘만들어놓으면 변경할 곳이 적으니까!
